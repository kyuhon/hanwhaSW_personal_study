세션 하이재킹(Session Hijacking)이란?
세션 하이재킹은 사용자의 세션 ID를 탈취해서 권한을 가로채는 공격입니다. 사용자의 로그인 세션을 공격자가 도용하면, 피해자의 권한으로 시스템에 접근할 수 있습니다.

📌 세션(Session)이란?
로그인 같은 상태 정보를 유지하기 위해 서버가 사용자에게 부여하는 고유 식별값 (세션 ID).

세션 ID는 보통 **쿠키(cookie)**에 저장되어 클라이언트 → 서버로 전달됨.

🛠️ 세션 하이재킹 방식 (종류)
공격 방식	설명
1. 세션 스니핑	네트워크에서 패킷 가로채기. HTTP에서 세션 ID 탈취 가능. (예: Wi-Fi)
2. XSS (크로스사이트 스크립팅)	악성 스크립트로 사용자의 쿠키에서 세션 ID 추출
3. 세션 고정(Session Fixation)	공격자가 세션 ID를 미리 지정해 사용자에게 쓰게 함
4. 브루트 포스	세션 ID가 예측 가능할 때, 무작위로 세션 ID를 추측 시도
5. 피싱	가짜 사이트로 유도 후 로그인 → 세션 정보 탈취

🧱 방어 방법 (예방 대책)
보안 방법	설명
✅ HTTPS 사용	세션 ID 전송 시 암호화로 스니핑 방지
✅ 세션 ID 자주 변경	로그인 후 세션 ID 재발급 → 세션 고정 방지
✅ HttpOnly 쿠키 설정	자바스크립트로 쿠키 접근 방지 → XSS 대응
✅ SameSite 쿠키 설정	CSRF 공격 방지
✅ 세션 타임아웃 설정	일정 시간 비활성 시 세션 자동 종료
✅ 사용자 IP/User-Agent 확인	세션이 다른 환경에서 사용되면 강제 종료
✅ XSS 필터링	입력값 검증 및 HTML 인코딩

🧨 실제 사례
과거에 카페24, 네이버 블로그 등에서 XSS를 통한 세션 탈취 사례 발생.

공공 Wi-Fi에서 HTTPS가 없던 웹사이트 이용 중 세션 스니핑 사례도 있었음.

🗒️ 요약 문장:
세션 하이재킹은 사용자의 세션 ID를 탈취해 인증을 우회하는 공격이며, 이를 방지하려면 HTTPS, 쿠키 보안 설정, 세션 관리 강화가 필수입니다.
