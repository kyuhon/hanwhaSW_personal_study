Session vs JWT Token
	Session의 단점 -> 반대가 JWT Token의 장점
		1. JSESSIONID유무
		2. 서버의 부하가 크다.(세션 클러스터링 - 별도의 저장소를 관리하여야 해서 인프라 증가)
	
JWT Token(feat. csrf)
저장되는 부분
	1. LocalStorage (뒷)
	2. Cookie (옆)
		1. SameSite
		  : Strict, Lax
		2. HttpOnly
		3. Https
	3. SPA의 반응형 변수
		Single Page Application
		react, reactNative
	
csrfToken에 jwtToken을 같이 들고 다니는 부분은 게이트웨이에서 csrf가 막히는 경우가 있다
그리고 csrfToken을 쓴다고 보안성이 그렇게 강화되지 않는다
